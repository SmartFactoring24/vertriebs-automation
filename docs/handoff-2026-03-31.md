# Handoff 2026-03-31

## Aktueller Stand

Der Bot kann auf dem Entwicklungsgerät inzwischen:

1. `smartclient.exe` starten oder an eine bestehende Session andocken
2. Login, 2FA, `VB-Portal`, `Mitteilungen` und das echte KI-Fenster automatisiert erreichen
3. innerhalb von `KI` bis `VBI -> Gruppen-Akte -> Eingereichtes Geschäft -> Einheiten nach Sparten der Gruppe` navigieren
4. den bereits geöffneten Zielzustand visuell erkennen und unnötige Klicks überspringen
5. den mittleren Tabellenbereich per Capture + OCR lesen
6. aus den OCR-Zeilen echte Archivdatensätze erzeugen
7. CSV- und XLSX-Snapshots lokal schreiben

Der aktuelle produktive Engpass liegt nicht mehr im KI-Datenpfad, sondern im WhatsApp-Web-Schritt auf dem Privat-PC. Der Archivlauf selbst funktioniert.

## Exportstatus

Der Archivteil funktioniert.

Aktueller Modus:

- keine timestamp-basierten Exportdateien mehr
- feste Snapshot-Dateien im Exportordner
- nur bei echten fachlichen Änderungen werden diese Dateien überschrieben
- wenn sich seit dem letzten Abtasten nichts geändert hat, passiert kein neuer Export

Aktuell relevante Dateien:

- `snapshot-current.csv`
- `snapshot-current.xlsx`

CSV ist jetzt mit UTF-8-BOM abgesichert, damit Umlaute in Excel sauber erscheinen.

Beispiel aus dem aktuellen Snapshot:

```csv
businessId,partnerStage,partnerName,productName,status,unitsValue,totalUnits,evaluationPeriod,businessScope,sourcePage,submittedAt,updatedAt,source
AL:Bo_hme_Erik:Leben,AL,"Böhme, Erik",Leben,eingereicht,64.8,70.8,04.2026,\VB- und VM-Geschäft,Einheiten nach Sparten der Gruppe im Eigengeschäft,...
```

Wichtige Restnotiz:

- Die aktuellen `snapshot-current.*` wurden einmal initial aus dem vorhandenen State erzeugt.
- Wenn der Exportordner auf einem neuen Gerät komplett leer ist und gleichzeitig noch keine fachliche Änderung vorliegt, muss entweder einmal ein Lauf mit Änderung stattfinden oder die Snapshot-Datei initial neu erzeugt werden.

## WhatsApp-Status

Der Bot kann WhatsApp Web selbst aufrufen, ist aber auf das **richtige Browserprofil** angewiesen.

Wichtig für die Praxis:

- Auf dem Privat-PC ist WhatsApp Web aktuell nicht mit der Office-Nummer im verwendeten Bot-Profil angemeldet.
- Deshalb schlägt der WhatsApp-Schritt dort erwartbar fehl.
- Dieser Fehler blockiert den Archivlauf jetzt **nicht mehr**.

Konkret:

- Archivdaten werden geschrieben
- ein WhatsApp-Fehler wird nur noch geloggt bzw. als Warnung ausgegeben
- der Einmallauf endet trotzdem sauber

Arbeits-PC-Hinweis:

- Wenn dort die Office-Nummer bereits täglich in WhatsApp Web genutzt wird, reicht es wahrscheinlich, das richtige Profil in `WHATSAPP_BROWSER_PROFILE` zu verwenden.
- Der Bot öffnet WhatsApp Web selbst über `WHATSAPP_WEB_URL`.

Zusätzlicher Schalter:

```env
WHATSAPP_NOTIFICATIONS_ENABLED=true
```

Für reine Archivtests oder Privatgeräte kann dieser Wert lokal auf `false` gesetzt werden.

## `.env` auf neuen Rechner übernehmen

Wichtig: Die echte `.env` liegt nur lokal vor und wird nicht gepusht. Auf einem neuen Rechner muss sie manuell angelegt werden.

Empfohlener Weg:

1. Repo auf dem Zielrechner klonen
2. `.env.example` nach `.env` kopieren
3. Werte lokal eintragen
4. insbesondere diese Punkte prüfen:

- `KI_APP_PATH`
- `KI_PASSWORD`
- `KI_MAIN_WINDOW_TITLE_HINT`
- `WHATSAPP_BROWSER_PROFILE`
- `WHATSAPP_NOTIFICATIONS_ENABLED`

## Minimiert-/Restore-Verhalten

Wenn `KI` minimiert ist:

1. Fenster wird wiederhergestellt/maximiert
2. Navigation/Auslese wird durchgeführt
3. danach wird wieder minimiert

Wenn `KI` bereits offen ist, bleibt es offen.
