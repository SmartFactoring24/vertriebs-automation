# Handoff 2026-03-31

## Erreichtes Zwischenziel

Der Bot kann den kompletten Desktop-Startpfad des DVAG-Clients auf diesem Gerät automatisiert durchlaufen bis zur geöffneten `KI`-Anwendung:

1. `smartclient.exe` starten
2. Login-Fenster `DVAG Online-System` erkennen
3. Passwort automatisch eintragen und Login absenden
4. auf die manuelle 2FA-Freigabe warten
5. erstes News-Fenster per Tastatur schliessen
6. `VB-Portal - Moritz Rolle` erkennen
7. `KI öffnen` im Portal anklicken
8. zweites News-Fenster `Mitteilungen` erkennen und schliessen
9. echtes Hauptfenster `Leo Mitteneder - DVAG Online-System` erkennen

Der finale erfolgreiche Verifikationstest lieferte:

```json
{
  "visibleWindowTitles": [
    "VB-Portal - Moritz Rolle",
    "Leo Mitteneder - DVAG Online-System"
  ],
  "mainWindowDetected": true,
  "stage": "main"
}
```

## `.env` auf neuen Rechner übernehmen

Wichtig: Die echte `.env` liegt **nur lokal** vor und wird wegen `.gitignore` **nicht** nach GitHub gepusht. Auf einem neuen Rechner muss sie deshalb manuell neu angelegt oder lokal übertragen werden.

Empfohlener Weg:

1. Repo auf dem Zielrechner klonen
2. `.env.example` nach `.env` kopieren
3. die unten stehende Vorlage in `.env` einfügen
4. nur noch die geheimen Werte oder gerätespezifischen Werte prüfen bzw. anpassen

## Wichtige Einschränkungen

- Die Benutzerkennung wird auf diesem Gerät nicht eingegeben, sondern ist bereits im Dropdown ausgewählt.
- Der derzeitige Portal-Klick auf `KI öffnen` ist fensterrelativ berechnet und für dieses Gerät validiert.
- `VB-Portal` bleibt im Hintergrund offen, während `KI` bereits geöffnet ist.
- Das echte Auslesen der Fachdaten aus `KI` ist noch nicht umgesetzt.
- Das WhatsApp-Web-Posting ist strukturell vorbereitet, aber noch nicht gegen die reale Gruppe verdrahtet.

## Reproduzierbarer Test

Im Repo-Ordner:

```powershell
npm run dev -- --login-ki
```

Der Test erwartet:

- lokales Windows-GUI
- 2FA-Freigabe auf dem Smartphone
- keine manuelle Interaktion während des Laufes

Für einen einfachen Start per Doppelklick liegt außerdem ein Alpha-Launcher bereit:

- `launcher/StartKiAutomation.exe`

Er ruft intern denselben Flow auf wie `npm run dev -- --login-ki`.
Er verwendet bevorzugt eine global installierte `Node`/`npm`-Version und fällt nur dann auf die portable Repo-Version unter `tools/` zurück.
Vor dem Start prüft der Launcher außerdem, ob `.env` und `node_modules` vorhanden sind.
Während eines laufenden Startvorgangs kann jederzeit `F12` gedrückt werden, um Bot und KI-Prozesse sauber abzubrechen.
